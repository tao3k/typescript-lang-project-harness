# CLI

The package exposes a thin command-line adapter over the default project
harness runner:

```shell
typescript-project-harness [--json] [--agent-snapshot] [PROJECT_ROOT]
```

When `PROJECT_ROOT` is omitted, the current working directory is used.

## Output Modes

Compact text is the default output for humans and repair-oriented agents:

```shell
typescript-project-harness .
```

Use `--json` when a tool needs the structured report payload:

```shell
typescript-project-harness --json .
```

Use `--agent-snapshot` when an agent needs the project reasoning tree before it
chooses an edit surface:

```shell
typescript-project-harness --agent-snapshot .
```

The snapshot is compact text, not JSON. It follows the Rust harness agent
snapshot shape: `Modules:`, `OwnerBranches:`, `OwnerDependencies:`, and
`FindingGroups:`. It includes module roles, exports, package entry owners,
package-name import owners, project-reference/workspace owner provenance, and
TypeScript-native import resolution, including type-only edges, so repair
agents can orient before editing. Type-only edges come from native TypeScript
syntax, including declaration-level imports, specifier-level `type` markers,
and `import("...")` type queries. Rust-style source-shape counters such as
`shadowed=` and `orphaned=` are emitted only when non-zero. The format design,
section order, line shapes, and non-goals are defined in
[Compact Agent Snapshot](204_compact_agent_snapshot.md).
When root parser facts expose TypeScript project-reference packages or
workspace packages, the CLI renders each non-empty package scope with a short
`pkg <path>` heading. Each member package is run from its own `package.json`
anchor and local `tsconfig.json`; it does not inherit the root package config.
The single-package compact format is covered by
`tests/snapshots/agent_snapshot_project.snap`; the project-level workspace
format is covered by
`tests/snapshots/agent_snapshot_workspace_project.snap`.
When native TypeScript diagnostics exist, the snapshot groups their findings
under `FindingGroups:`. Full diagnostic cards, native TypeScript codes such as
`TS2322`, and related diagnostic information remain in the default compact
renderer and JSON output.
When TypeScript selects JavaScript via `allowJs`, JavaScript owners and edges are
rendered from the same parser-owned project report.

## Exit Codes

- `0`: no configured-blocking findings
- `1`: configured-blocking findings exist
- `2`: CLI argument or project-root error

The CLI delegates to library APIs. It does not own TypeScript parsing,
workspace discovery semantics, or project-specific policy.
