# CI

GitHub Actions runs the same package-owned validation surface as local
development:

```shell
npm ci
npm run check:implementation
npm run check:policy
npm run lint
npm run format:check
npm run test:implementation
npm run test:policy
npm run harness
git diff --check
```

The workflow uses the repository `package-lock.json`, Node 24, npm cache, and
the npm-pinned `oxlint` and `oxfmt` binaries so CI does not rely on a developer
shell.

`npm run check:implementation` owns TypeScript implementation validity through
`tsc --noEmit`. `npm run check:policy` owns the harness self-policy gate.
`npm test` is composed from `test:implementation` and `test:policy`, keeping
compiler/test failures separate from policy advice.

Downstream packages can use `ts-harness check --full .` from `package.json`
scripts when a shell command should surface agent repair instructions without
adding a project-local runner script.
